# StudyOS Projects Feature - Complete Guide

## Overview

The Projects feature is a comprehensive project management system designed specifically for students. It combines file management, documentation, task tracking, and version control integration into a single powerful platform.

## Main Features

### 1️⃣ Project Overview Dashboard
When you open a project, you'll see:
- **Project Status**: Ongoing, Submitted, or Completed
- **Subject/Module**: Track which course this belongs to
- **Priority Level**: High, Medium, or Low
- **Deadline Countdown**: See days remaining until submission
- **Progress Bar**: Visual representation of task completion
- **Storage Usage**: Monitor your file storage quota (100 MB limit)

Link: Navigate Projects → Click "View Details" on any project card

### 2️⃣ File Manager & Study Materials
Upload and organize all your study materials in one place.

**Supported Formats:**
- Documents: PDF, DOCX, Word files
- Presentations: PPT, PPTX
- Images: JPG, JPEG, PNG, GIF
- Archives: ZIP, RAR files

**Folder Organization:**
- **Notes**: Class notes, lecture slides
- **Assignments**: Assignment materials and guidelines
- **Resources**: Additional reference materials
- **Submissions**: Your submitted work

**File Tagging:**
Tag files as: Exam, Assignment, Revision, or Reference for quick filtering

**Storage Tracking:**
- Visual progress bar shows storage usage
- 100 MB limit per project
- File size displayed for each upload

### 3️⃣ Code Workspace
Save and organize code snippets for quick reference.

**Features:**
- Support for multiple languages (JavaScript, Python, Java, C++, SQL, React, TypeScript, HTML, CSS, NodeJS)
- One-click copy to clipboard
- Search and filter snippets
- Add descriptions for context
- Keep reusable code patterns handy

**Example Workflow:**
1. Create a snippet for "Firebase Authentication Helper"
2. Add description and code
3. Select language: JavaScript
4. Later, quickly copy and paste when needed

### 4️⃣ GitHub Integration
Connect your GitHub repository and track development progress.

**What You Can See:**
- **Repository Info**: Direct link to your GitHub repo
- **Branches**: View all branches with last update time
- **Commits**: See recent commits with author, message, and timestamp
- **Quick Links**: Open repository, view specific commits, browse branches

**Connection:**
1. Go to GitHub tab
2. Click "Connect Repository"
3. Paste your GitHub URL
4. System auto-syncs commit history

### 5️⃣ SRS & Documentation
Create and manage technical documentation for your projects.

**Supported Documents:**
- System Requirements Specification (SRS)
- Project Reports
- Research Notes
- Design Documents
- API Documentation

**Features:**
- **Rich Markdown Editor**: Format with headers, bold, italic, links
- **Version History**: Track document revisions (v1, v2, v3...)
- **Timestamps**: See when each version was created
- **Full Preview**: View complete documentation

**Workflow:**
1. Click "Create Documentation"
2. Add title (e.g., "System Requirements Specification")
3. Write content using Markdown
4. Updates increment version number automatically

### 6️⃣ Bug & Issue Tracker
Track bugs and issues found during development.

**Issue Details:**
- **Title & Description**: What needs to be fixed
- **Severity**: Critical, High, Medium, Low
- **Status**: Open → In Progress → Fixed → Closed
- **Timestamps**: When issue was reported

**Workflow:**
1. Click "Report New Bug"
2. Enter issue title and description
3. Set severity level
4. Initial status set to "Open"
5. Update status as you work on it

### 7️⃣ Submission Tracker (Unique Feature!)
Track all project submissions with version history.

**Perfect For:**
- Multiple assignment submissions
- Versioned project deliverables
- Keeping accountability
- Submission dates and history

**Features:**
- **Version Control**: v1, v2, v3... automatically tracked
- **Submission Dates**: Know exactly when you submitted
- **File Management**: Download any previous version
- **Status Tags**: Mark each submission as "Submitted"

**Workflow:**
1. Click "New Submission"
2. Add title (e.g., "Assignment 1 - Final", "Project Report v2")
3. Select file to upload
4. System tracks version number automatically

### 8️⃣ Task Management (Kanban Board)
Visual task management with three columns.

**Columns:**
- **To Do**: Tasks to start
- **In Progress**: Currently working on
- **Done**: Completed tasks

**Task Details:**
- Title and description
- Due date (optional)
- Priority: Low, Medium, High
- Drag or button to move between columns

**Quick Actions:**
- Click buttons to move task to different columns
- Delete completed tasks
- Set priority and due date when creating

### 9️⃣ Notes & Quick Thoughts
Quick note-taking space for ideas and thoughts.

**Features:**
- Yellow "sticky note" style cards
- Add title and content
- Search notes
- Edit anytime
- Delete when no longer needed

**Use Cases:**
- Jot down ideas while coding
- Keep learning notes
- Todo items
- Quick reminders

### 🔟 Activity History
See everything that happens in your project.

**Tracked Activities:**
- File uploads
- Documentation created/updated
- Tasks created/moved
- Submissions uploaded
- Bugs reported
- Notes added
- Access level changes

**View:**
- Click "Activity" tab
- See timeline of all events
- Timestamps for each activity
- Summary statistics

### 1️⃣1️⃣ Access Control
Control who can access your projects.

**Levels:**
- **Private** (Default): Only you can see
- **View Only**: Others can view but not edit
- **Can Edit**: Others can make changes

**How to Change:**
1. Click share icon in project detail
2. Select access level
3. Done! Activity logged automatically

### 1️⃣2️⃣ Project Stats & Overview
See project metrics at a glance.

**Dashboard Shows:**
- Total projects
- Active/ongoing projects
- Completed projects
- Total deployed tasks
- Progress percentages
- Storage usage per file

## Getting Started

### Create a New Project

1. Click **"New Project"** button in Projects page
2. Fill in project details:
   - **Project Name**: e.g., "E-Commerce Platform"
   - **Subject/Module**: e.g., "Database Management" or "DBMS"
   - **Tech Stack**: e.g., "React, Firebase, Tailwind"
   - **GitHub Repo**: (Optional) GitHub repository URL
   - **Status**: Ongoing, Submitted, or Completed
   - **Priority**: High, Medium, or Low
   - **Deadline**: When it's due
   - **Description**: What is this project about?

3. Click **"Deploy Project"** to create
4. Now use all the tabs to manage your project!

### Navigate Between Projects

- **View all projects**: Click back arrow in project detail
- **Search projects**: Use search bar on main page
- **Filter by status**: Click filter buttons (All, Active, Completed, etc.)

### Typical Workflow

**Week 1: Setup**
1. Create project with details
2. Upload initial files (assignment, slides, etc.)
3. Add GitHub repository link
4. Create task list for deliverables

**Week 2-3: Development**
1. Update documentation as you build
2. Add code snippets for reference
3. Track tasks on Kanban board
4. Report and fix bugs
5. Add quick notes and ideas

**Week 4: Submission**
1. Create final submission
2. Upload to submissions tab
3. Track version number
4. Check activity log for confirmation
5. Mark tasks as complete

## Tips & Tricks

🎯 **Organization Tips:**
- Tag files consistently (Exam, Assignment, etc.)
- Use descriptive task titles
- Keep documentation updated
- Version your submissions clearly

📱 **Mobile-Friendly:**
- Stack layout works on phones
- Touch-friendly buttons
- All features available mobile

⚡ **Performance:**
- Storage is limited - delete old files
- Keep projects focused on one deliverable
- Archive completed projects

🔒 **Privacy:**
- Default private (only you see)
- Share selectively with groupmates
- Change access level anytime

## Storage Quota

- **Limit**: 100 MB per project
- **Tracking**: Visual bar shows usage
- **What to Do if Full**:
  1. Delete old/unnecessary files
  2. Archive submissions to external storage
  3. Compress files before uploading

## Data Storage

All project data is saved to:
- **Browser Storage**: For quick access
- **Firebase**: For reliable backup (when integrated)

## Keyboard Shortcuts

Some tips for efficiency:
- **Search**: Ctrl/Cmd + K to open search
- **Create**: Click "+" buttons throughout
- **Edit**: Hover over items to reveal edit/delete

## Troubleshooting

**Problem**: File upload fails
- **Solution**: Check file size and format

**Problem**: GitHub repo not syncing
- **Solution**: Verify URL is correct, try reconnecting

**Problem**: Storage full
- **Solution**: Delete old files or download them elsewhere

**Problem**: Submissions not showing
- **Solution**: Refresh page, check if uploaded successfully

## Best Practices
